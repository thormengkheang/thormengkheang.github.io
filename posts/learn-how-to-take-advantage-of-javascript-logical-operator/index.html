Take advantage of logical operator.

Table of Contents:

Today we will explore how to take advantage of logical operator AND (&&) and OR (||).

Logical Operator OR (||)

In Javascript logical operator OR (||) return true if either operand is true.

true || true // true
true || false // true
false || false // false

Javascript logical operator they evaluate from left to right when they evaluate an OR (||) expression if the first operand is true, it will not continue to evaluate the second operand or any other operand after that. It means they short-circuit. So this mean the operand after the first operand can be of any value.

true || 'any value' // true

Because of this we can use them to our advantage! For example if person object has property of name and age and we want to console.log job property but we don’t know whether the object has job property or not.

var person = {
    name: 'james',
    age: 16
};

console.log(person.job || 'unemployed'); // unemployed

In the example above we console.log the value of person.job OR the default string of unemployed Since person.job doesn’t exist it return undefined Since undefined is a falsy value, Javascript will instead evaluate the value of the other side of the || and return whatever value there.

Let look what happen when person object has job property:

var person = {
    name: 'james',
    age: 16,
    job: 'writer'
};

console.log(person.job || 'unemployed'); // writer

In this time person.job exists. So the OR (||) expression return value of writer is logged to the console instead.

One more example to test what we’ve learned so far. This is a popular snippet of code from Stackoverflow:

var a;
var b = null;
var c = undefined;
var d = 4;
var e = 'five';

var f = a || b || c || d || e;

console.log(f);

Try to answer what would be logged to the console. Scroll down for the answer…

Is your answer 4? Good job if not, that’s okay too. Let break it down:

var a; // undefined (falsy value)
var b = null; // null (falsy value)
var c = undefined; // undefined (falsy value)
var d = 4; // number (NOT falsy)
var e = 'five'; // assigment short circuits before reaching here

var f = a || b || c || d || e;

All a, b, c are falsy value. When logical OR (||) return a falsy value, it continue to evaluate the next operand. When it reached d with value of 4 it will return the value 4 and stop evaluate the next operand and the value of 4 is assgined to varaible f.

Logical Operator AND (&&) with React

This is section we will explore how to take advantage of logical operator AND (&&) with React or React Native. When working with React we often need to show and hide element give certian condition. This is called conditional rendering. We mostly use IF for this approach:

const MyComponent = ({ loading, data }) => {
    if (loading) {
        return <p>loading</p>;
    }

    return (
        <ul>
            {data.map(item => <li>{item.name}</li>)}
        </ul>
    );
};

There is nothing wrong with the example above, on the contrary, code is simple and readable. However, this approach is restricted to render a whole block in most of the case, we can not use it.

Imagine our component is more complex that it has a title and a button to create new item. The page should alway show the title and button. Loading message would only appear in the place of the list:

Considering the image above, we can not use IF if we use IF, we would repeat the entire block just the change the content of list.

const MyComponent = ({ loading, data }) => (

In the example, we use braces ({}) to insert Javascript expression in JSX. React will evaluate each expression and render it. React will not render anything it just ignores if the expression return boolean, undefined or null.

React Native will not ignore undefined it will throw an runtime error. Just to be safe just convert the value boolean if you’re working outside of the browser.

To understand the example above, according to MDN logical operator AND (&&)

expr1 && expr2

Returns expr1 if it can be converted to false; otherwise, returns expr2. Thus, when used with Boolean values, && returns true if both operands are true; otherwise, returns false.

This mean Javascript will evaluate each expression until it reaches the end or it short-circuit when evaluation end up in a falsy value and the last value of the evaluation will be the result of the expression. See the example below:

const person = {
    name: 'james',
    address: null,
};

console.log(person && person.name) // james
console.log(person && person.address) // null
console.log(person && person.address && person.address.zipcode // null

In the example expression person && person.name since the first expression person is a truthy value, it continue to the second expression person.name since it is the last expression, the result of expression will be the value of the second expression james.

In case of address the first expression is also a truthy value then the value of person.address is null a falsy value since it is also the last expression, it becomes the result of the expression.

Finally in case of zipCode the same process as the address occur. It evaluate the first expression then the second. However the evaluation of the second expression is a falsy value so Javascript short-circuit and stop evaluate the next expression, as alway the last expression to be evaluated will be become the result of the expression. (If Javascript did not stop the expression it will throw an error.)

It is important to understand how the expressions work, because sometimes you may be expecting the result to be false, and then you get null or 0. Especially when it is 0, React will render it as 0 unlike boolean, undefined or null.
